Quote exposes its analytics as an [MCP](https://modelcontextprotocol.io) server, so AI agents ‚Äî Claude, or anything that speaks MCP ‚Äî can query markets and your account directly. The connector is **strictly read-only**: it never places, modifies, or cancels orders.

## OAuth flow (remote connector)

The `/mcp` endpoint uses OAuth 2.1 bearer tokens ‚Äî not Privy or HMAC. Standards-compliant discovery means most MCP clients connect with just the URL:

| Endpoint | Role |
|---|---|
| `/.well-known/oauth-protected-resource` | Resource metadata (RFC 9728) ‚Äî points clients at the authorization server |
| `/.well-known/oauth-authorization-server` | Authorization-server metadata (RFC 8414) |
| `POST /oauth/register` | Dynamic client registration (RFC 7591) |
| `GET /oauth/authorize` | Redirects to Quote's consent page |
| `POST /oauth/token` | Authorization-code + refresh-token grants, with PKCE |

An unauthenticated call to `/mcp` returns `401` with a `WWW-Authenticate` header pointing at the metadata ‚Äî which is how clients bootstrap the flow automatically.

## Scopes

| Scope | Grants |
|---|---|
| `analytics:read` | Market-wide data: prices, candles, funding history, order-book microstructure, market screeners, news, earnings, prediction markets, pre-trade cost estimation |
| `account:read` | Your wallet's data: execution report, portfolio report, funding carry, triggers, fee tier |

A token only ever sees the account it was issued for, and tool listings are filtered to the token's scopes ‚Äî an `analytics:read`-only token doesn't even see account tools.

## Available tools

The catalog (with input schemas) is always current at `GET /api/mcp/tools`; `GET /api/mcp/guide` returns an operating guide written for agents. Highlights:

| Tool | Scope | What it returns |
|---|---|---|
| `get_price`, `get_candles` | `analytics:read` | Live prices and OHLCV history |
| `get_market_microstructure` | `analytics:read` | Book depth, spread, imbalance signals |
| `estimate_trade_cost`, `stress_test` | `analytics:read` | Pre-trade cost / liquidation stress estimates |
| `scan_markets`, `search_news`, `get_earnings`, `search_outcome_markets` | `analytics:read` | Screeners, news, earnings, prediction markets |
| `get_execution_report`, `get_portfolio_report` | `account:read` | Your execution quality and portfolio analytics |
| `get_funding_carry`, `get_funding_history` | `account:read` | Funding economics for your positions |
| `list_triggers`, `get_fee_info`, `get_referral_summary` | `account:read` | Your triggers, fee tier, referral status |

### Symbol formats

Symbols are venue-prefixed, and common names are resolved for you (`"oil"` ‚Üí `xyz:CL`):

- Perps: `BTC`, `ETH`, `HYPE`
- Builder-DEX equities/commodities: `xyz:HOOD`, `xyz:CL`
- Prediction markets: `#2290` (HIP-4 outcome coins)

### Conventions

Execution analytics follow the platform-wide [benchmark convention](/guides/analytics#the-benchmark-convention): signed bps, positive = worse. Prices are live snapshots and may differ slightly from the terminal's WebSocket-fed views.

## Dispatching a tool over REST

```json
POST /api/mcp/dispatch
{
  "tool": "get_price",
  "arguments": { "symbol": "ETH" }
}
```

Returns `{ "result": ‚Ä¶ }` on success or `{ "error": "‚Ä¶" }` on tool failure. Scope enforcement applies to API-key callers per tool.
