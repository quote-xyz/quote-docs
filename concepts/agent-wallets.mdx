---
title: "Agent Wallets"
description: "How Quote trades on your behalf without ever holding your funds."
---

Quote is **non-custodial**. It never holds your keys or your funds. Instead, trading happens through Hyperliquid's native agent-wallet mechanism: a separate secp256k1 key that can *sign orders* for your account but can **never withdraw**.

## The model

| | Master wallet (yours) | Agent wallet (Quote-managed) |
|---|---|---|
| Holds funds | ✅ | ❌ |
| Can withdraw | ✅ | ❌ |
| Can place/cancel orders | ✅ | ✅ (after you approve it) |
| Key location | Your wallet / Privy embedded wallet | Generated by Quote, encrypted at rest (XChaCha20-Poly1305) in an isolated key vault |

You authorize the agent **once**, by signing Hyperliquid's `approveAgent` action with your master wallet. From then on, Quote signs orders with the agent key. That is why algo strategies can work an order for hours without asking you to sign every child slice.

## Lifecycle

<Steps>
  <Step title="Create">
    `POST /api/agents` generates a fresh agent keypair server-side. The private key is encrypted at rest; the response returns only the public `agentAddress`.
  </Step>
  <Step title="Approve on Hyperliquid">
    Your master wallet signs an `approveAgent` action over the agent address and a nonce (a millisecond timestamp). The terminal handles this signature; then `POST /api/agents/register` submits it to Hyperliquid.

    ```json
    {
      "agentAddress": "0x…",
      "signature": "0x…",
      "nonce": 1752402000000,
      "agentName": "quote-terminal"
    }
    ```
  </Step>
  <Step title="Approve the builder fee (required to trade)">
    Every order routed through Quote carries Quote's builder code (see [Hyperliquid constraints](/concepts/hyperliquid-constraints#builder-fee)), and **Hyperliquid rejects orders from wallets that haven't approved the builder fee**. This step is mandatory, not optional. Your master wallet signs the `approveBuilderFee` action once on Hyperliquid; `POST /api/agents/builder-approval` records it (Hyperliquid remains the source of truth). Until it is done, every order submission fails, whether from the terminal or the API.
  </Step>
  <Step title="Trade">
    Every order submission resolves your wallet's current agent, verifies its registration against Hyperliquid, and signs the exchange action with it.
  </Step>
</Steps>

<Note>
  Steps 2 and 3 require signatures from your **master wallet**, which only a terminal session can produce. API-key clients can create and read agents (`agents:write` / `agents:read`) but the initial approval happens in the [terminal](https://quotemarkets.xyz).
</Note>

## Signing flow

When Quote signs an action on your behalf:

1. Decrypt the agent private key (XChaCha20-Poly1305 authenticated encryption).
2. Compute the EIP-712 typed-data hash of the Hyperliquid action.
3. Sign with secp256k1 → `(r, s, v)`.
4. Zeroize the key material in memory.

Before signing, Quote verifies with Hyperliquid that the agent is still registered **and belongs to your wallet**: `userRole(agent_address)` must return `role == "agent"` with `data.user` equal to your normalized wallet address. A stale or revoked agent fails closed.

## Checking agent status

```bash
GET /api/agents
```

```json
{
  "success": true,
  "agentAddress": "0x…",
  "createdAt": 1752400000,
  "expired": false,
  "builderFeeApproved": true,
  "termsAcceptedAt": 1752400000,
  "termsVersion": "1.0"
}
```

- A missing/unregistered agent returns `success: true` with no `agentAddress`.
- An expired agent returns `expired: true`. Re-run the approval flow in the terminal.

## Revoking

You can revoke an agent at any time directly on Hyperliquid (it's your account). Quote fails closed: a revoked agent can't sign, and order submission returns an error rather than falling back to anything else.
